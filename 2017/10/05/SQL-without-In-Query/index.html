怎么躲开SQL中的in操作？

为什么要避免在SQL中的使用IN操作？我们先从IN操作是一个什么样的原理，在数据库做了些什么样的操作？

从上面就能看出，所谓的IN操作就是两张表的全连接。这里有一个问题，我们查询SQL需要的是TABLE1的数据，但是IN操作把TABLE2的数据全部遍历一遍，但这只是为了某个字段。
所以当TABLE2的内容非常大的时候，IN操作就显得有点慢了。那怎么解决这个问题呢？答案是使用EXISTS的操作替换IN操作。
